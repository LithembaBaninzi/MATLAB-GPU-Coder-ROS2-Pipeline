# generated/

Auto-generated artefacts produced by MATLAB GPU Coder and ROS2 package generation scripts.

> **Do not edit files in this folder by hand.** They are overwritten every time a build script in `matlab/codegen_cfg/` is run. Make changes in `matlab/algorithms/` or `matlab/codegen_cfg/` instead, then re-run the build.

---

## Sub-folders

### `cuda_lib/`

Compiled CUDA shared libraries and MEX files, ready to be linked into ROS2 packages or standalone executables.

| File | Source algorithm | Used by |
|---|---|---|
| `nanoP3p.so` | `matlab/algorithms/nanoP3p.m` | `visStateEst3` ROS2 node |

To regenerate:
```matlab
compile_nanoP3p   % run in matlab/codegen_cfg/
```

---

### `tgz_files/`

Compressed ROS2 package archives (`.tgz`) ready to be copied to the Jetson and unpacked into `~/ros2_ws/src/`.

| File | Source build script | ROS2 node inside |
|---|---|---|
| `invertCameraRos2.tgz` | `buildInvertCamRos2.m` | `invertCameraRos2` |
| `updatedVisStateEst.tgz` | `compile_visStateEst.m` | `visStateEst3` |

Deploy to Jetson:
```bash
scp generated/tgz_files/<package>.tgz user@jetson:~/
# On Jetson:
tar -xzf <package>.tgz -C ~/ros2_ws/src/
cd ~/ros2_ws && colcon build && source install/setup.bash
```

---

### `CMakeLists/`

CMake build files emitted by MATLAB Coder.

| File | Notes |
|---|---|
| `CMakeLists.txt` | Original file generated by MATLAB — do not modify. |
| `CMakeLists_Updated.txt` | Working version with fixes applied: correct GStreamer package detection, removed problematic MATLAB library import, cleaned compile options, added proper library directories. Use this one when rebuilding on Jetson. |

If a Jetson build fails, compare your current `CMakeLists.txt` against `CMakeLists_Updated.txt` to find the issue.

## Regenerating Everything

Run these scripts in order from `matlab/codegen_cfg/` on the host PC:

```matlab
% 1. CUDA shared libraries
buildGpuGrayLib
buildGpuInvertLib
compile_nanoP3p

% 2. ROS2 package archives
buildJetsonGray
buildInvertCamRos2
compile_visStateEst
```
